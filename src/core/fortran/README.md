# core

Ce dossier contient les applications internes, prototypes et démonstrations avancées du SDK.

Ces fichiers ne font pas partie de l’API publique.  
Ils servent principalement à :

- tester les modules et l’API haut niveau
- valider le comportement multiplateforme
- démontrer des fonctionnalités complexes
- expérimenter de nouvelles idées

Pour des exemples simples destinés aux utilisateurs, voir le dossier `examples/`.
